# Внесение изменений

1. Измените семантику маршрутизации в `policy/`, клиентские настройки в
   `targets/`, а пользовательские geosite-списки в `data/`.
2. Установите `requirements.txt` и выполните `make configs && make test`.
3. Для полной проверки с сетевым доступом выполните `make build`.
4. Откройте Pull Request и дождитесь обеих CI jobs.

Не добавляйте в репозиторий содержимое `dist/`: бинарные файлы публикуются как
release assets. Для доменов используйте минимально необходимое правило и кратко
объясняйте причину изменения в Pull Request.

Генераторы могут только преобразовывать валидированную модель в формат клиента.
Приоритет, fallback, классификация и другая бизнес-логика принадлежат `policy/`
или `routing_engine/model.py`.
